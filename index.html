Introduction

Parmi la multitude d'outils que doit savoir manipuler un développeur, il y a des incontournables: les environnements de développement (IDE). Mais lorsqu'on débute il n'est pas aisé de s'y retrouver pour savoir sur lequel s'attarder principalement: langages favorisés, fonctionnalités, design, extensions, raccourcis d'édition... Autant de points à considérer que d'environnements à disposition, de quoi faire tourner la tête. Longuement débattues sur moult forums, les qualités de l'un ou l'autre seront malmenées par tel ou tel partisan pur et dur, mais même à travers les préférences personnelles de chacun il se dégage des points saillants. En attendant voici un aperçu supposément objectif de 5 d'entre eux, parmi les plus utilisés sur le marché.

Environnements de développement

IntelliJ IDEA

  1. Caractéristiques principales:

    1. Langages

      Variés, emphase sur Java, Kotlin, Groovy & Scala.

    2. Debug

      Points d'arrêt, fenêtres espion, options pas à pas, débogage à distance. Débogueur natif pour tous les langages cités ci-dessus.

    3. Contrôle de version

      Git, SVN, et Mercurial intégrés.

    4. Fonctionnalités

      Autocompletion avancée, suggestions de code, profilage, support de bases de données, affichage SVG intégré.

    5. Plugins

      Très riche via l'IntelliJ Plugin Repository.

  2. Points forts:

    • Interface utilisateur intuitive
    • Large écosystème de plugins
    • Analyse de code avancée
    • Support pour de nombreux langages de programmation

  3. Points faibles:

    • Consommation de ressources élevée
    • Peut être coûteux pour la version Ultimate

Eclipse

  1. Caractéristiques principales:

    1. Langages

      Java, plus avec plugins.

    2. Debug

      Points d'arrêt, fenêtres espion, options pas à pas, débogage à distance.

    3. Contrôle de version

      Pas intégré de base, mais plugins officiels pour Git et SVN.

    4. Fonctionnalités

      Très extensible, autocompletion, suggestions de code.

    5. Plugins

      Vaste collection de plugins via Eclipse Marketplace, moins facile à configurer que pour autres IDE.

  2. Points forts:

    • Open-source et gratuit
    • Personnalisable grâce aux plugins

  3. Points faibles:

    • Interface utilisateur moins intuitive que d'autres IDE
    • Terminal non intégré de base
    • Peut être lent avec de nombreux plugins installés
    • Interface vieillissante

Visual Studio

  1. Caractéristiques principales:

    1. Langages

      Variés, emphase sur C#, C++, F#, et Java script.

    2. Debug

      Points d'arrêt, fenêtres espion, options pas à pas, CallStack check.

    3. Contrôle de version

      Git et TFVC intégrés.

    4. Fonctionnalités

      Profilage, support de bases de données, autocompletion, suggestions de code, affichage SVG intégré.

    5. Plugins

      Via Extension Marketplace.

  2. Points forts:

    • Interface utilisateur moderne et intuitive
    • Large écosystème de plugins et d'extensions

  3. Points faibles:

    • Consommation de ressources élevée
    • Peut être coûteux pour certaines versions

PyCharm

  1. Caractéristiques principales:

    1. Langages

      Python et langages web.

    2. Debug

      Débogage à distance, points d'arrêt, fenêtre espion, suggestions d'optimisation.

    3. Contrôle de version

      Git intégré.

    4. Fonctionnalités

      Profilage, support de bases de données, autocompletion, suggestions de code.

    5. Plugins

      Via JetBrains Plugin Repository.

  2. Points forts:

    • Interface utilisateur intuitive
    • Support pour le développement web avec Django et Flask

  3. Points faibles:

    • Consommation de ressources élevée
    • Peut être coûteux pour la version Professional

Visual Studio Code

  1. Caractéristiques principales:

    1. Langages

      Langages web, Python, et plus si plugins.

    2. Debug

      Points d'arrêt, fenêtres espion, options pas à pas. Nécessite plus de set-up qu'un IDE.

    3. Contrôle de version

      Git intégré.

    4. Fonctionnalités

      Léger, très personnalisable, très extensible, autocompletion, suggestions de code.

    5. Plugins

      Immense Marketplace d’extensions.

  2. Points forts:

    • Léger et rapide
    • Personnalisable grâce aux extensions
    • Bien pour les débutants

  3. Points faibles:

    • Moins de fonctionnalités avancées par rapport à des IDE complets
    • Nécessite des extensions pour certaines fonctionnalités

En bref

Forces Faiblesses

- Bon support Maven, Gradle, Spring
- Refactoring avancé
- Intégration Git solide
- Auto-complétion très efficace
- Version Community gratuite 		- Gourmand en ressources
- Interface complexe au départ
- Courbe d'apprentissage raide au début
- Version Pro chère
Eclipse - Mature, utilisé en entreprise
- Bon support Java (C/C++ aussi avec plugins)
- Très configurable
- Large écosystème de plugins
- Gratuit et open source 		- Interface vieillissante
- Prise en main difficile
- Plugins parfois instables
- Lent au démarrage
Visual Studio - Excellente intégration .NET/C#/C++
- Debugger puissant
- UI soignée
- Outils pro intégrés (tests, profiling, Azure...)
- Idéal pour le dev Windows
- Version Community gratuite 		- Très lourd (poids, interface)
- Pas natif sous Linux
- Trop complexe pour petits projets
- Surchargé hors .NET/C++
- Version payante très chère
PyCharm - IDE Python très complet
- Excellent debug/test/venv intégré
- Analyse de code intelligente
- Interface propre
- Version Community gratuite 		- Lourd comparé à VS Code
- Dense pour les débutants
- Lent au démarrage
- Moins adapté à d'autres langages
VS Code - Léger, rapide à lancer
- Très personnalisable via extensions
- Interface moderne et intuitive
- Excellente prise en charge de nombreux langages (via plugins)
- Parfait pour Git, Markdown, et le dev web.
- Courbe d'apprentissage douce
- Gratuit et open source 		- Beaucoup de choses à installer soi-même
- Moins puissant pour gros projets
- Debug moins fluide que dans des IDE complets

Conclusion

Un bon point de départ pour le choix d'un IDE sera avant tout votre langage d'intérêt. On peut ensuite peser la courbe d'apprentissage requise pour arriver à s'en sortir dans l'IDE. Une fois l'IDE choisi, dès l'acquisition d'un peu d'aisance on pourra fouiller un peu plus dans les options. Il pourra alors être pertinent d'aller tester les environnements concurrents une fois les compétences de base développées. En effet, les habitudes acquises sur l'un peuvent souvent se transférer sur l'autre, parfois en changeant quelques préférences, d'autres fois en allant chercher le bon plugin. De plus il n'est pas rare de trouver des fonctionnalités similaires ailleurs. Le tout c'est de commencer, alors à vos claviers !
